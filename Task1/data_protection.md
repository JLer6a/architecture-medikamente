# Категории данных и методы их защиты

## 1. Персональные данные (PII)

**Примеры:**
- ФИО
- Дата рождения
- Телефон, email
- Адрес проживания
- Паспортные данные

**Методы защиты:**
- Тег: `PII`
- Шифрование при хранении (AES-256)
- Шифрование при передаче (HTTPS, TLS 1.2+)
- Доступ только по ролям (RBAC/ABAC)
- Аудит доступа (журналирование)

---

## 2. Медицинские данные (PHI)

**Примеры:**
- Диагнозы
- Медицинская карта
- История обращений
- Результаты анализов
- Заключения врачей

**Методы защиты:**
- Тег: `PHI`
- Шифрование в хранилище и при передаче
- Строгое разграничение доступа (только лечащий врач)
- Логирование доступа
- Интеграции с лабораториями через защищённые API (OAuth2, JWT, Masking)

---

## 3. Финансовые данные

**Примеры:**
- Платёжная информация
- Суммы оплат
- Информация из 1С:Бухгалтерия

**Методы защиты:**
- Тег: `FIN`
- Доступ только кассирам и бухгалтерии (RBAC)
- Шифрование базы
- Обезличивание для BI-аналитики

---

## 4. Документы и файлы

**Примеры:**
- Скан анкеты пациента
- JPG, PDF, DOC файлы на файловом сервере
- Медицинские заключения, справки

**Методы защиты:**
- Тег: `SENSITIVE`
- Контролируемое файловое хранилище
- Ограничения доступа (только по ролям)
- Шифрование на уровне тома / файловой системы
- Мониторинг доступа (например, DLP)

---

## 5. Логи и технические данные

**Примеры:**
- Журналы входа в систему
- Аудит доступа
- Запросы к API

**Методы защиты:**
- Маскирование чувствительных данных
- Мониторинг на предмет аномалий
- Алерты при подозрительных действиях

---

## 6. Атрибуты доступа (контролирующие данные)

**Примеры:**
- ID сотрудника / пациента
- Роли и политики
- Метки безопасности (теги)

**Методы защиты:**
- Хранение в IAM-системе или в отдельной подсистеме управления доступом
- Использование ABAC: доступ по ID, роли, контексту
- Аудит изменений политик
